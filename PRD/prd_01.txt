제품 요구사항 정의서 (Product Requirements Document - PRD) v0.1

프로젝트 명: Spring 영어 원서 읽기 동아리 홈페이지
작성일: 2025년 4월 29일
버전: 0.1
작성자: 윤현지
1. 개요 (Introduction)

본 문서는 "Spring 영어 원서 읽기 동아리" 활동을 지원하기 위한 홈페이지 구축 프로젝트의 제품 요구사항을 정의합니다. 본 홈페이지는 동아리 회원들이 정보를 공유하고 소통하며, 모임 일정을 관리하고, 관리자가 회원을 효율적으로 관리할 수 있도록 하는 웹 기반 플랫폼을 제공하는 것을 목표로 합니다.

2. 목표 (Goals)

동아리 회원 및 잠재 회원에게 동아리 활동 정보를 투명하게 제공합니다.
회원 가입 및 관리를 시스템화하여 관리자의 업무 부담을 줄입니다.
회원 간의 정보 공유 및 소통을 활성화합니다.
모임 일정 및 자료를 효율적으로 관리하고 공유합니다.
정보 접근 권한을 구분하여 보안성을 확보합니다.

3. 대상 사용자 (Target Audience)

동아리 회원: 홈페이지의 대부분 기능을 이용하는 주 사용자. 게시물 열람/작성, 자료 열람, 사진 열람, 수다방 이용, 회원 연락처 조회 등을 수행합니다.
잠재 회원: 동아리에 관심 있는 사용자. 소개 페이지를 통해 정보를 얻습니다.
도서관 사서: 특정 관리 권한(모임 일정 등록/수정)을 가진 사용자 그룹. 회원 관리 권한은 없습니다.
관리자: 시스템 전반의 관리 권한을 가진 사용자 그룹. 회원 승인, 회원 정보 수정, 회원 구분 수정, 모임 일정 등록/수정 등을 수행합니다.

4. 범위 (Scope)

4.1. 포함 범위 (In-Scope)

회원 가입 기능 (이름, 아이디, 휴대폰 번호, 메일 주소 필수, 가입 암호 'test123')
관리자 회원 가입 승인 기능
회원 정보 관리 기능 (관리자와 회원 본인만 수정 가능)
회원 구분 (도서관 사서, 동아리 멤버, 관리자) 기능
로그인 기능 (일반 회원/사서는 비밀번호 'test123', 관리자는 'admin/admin')
비로그인 시 소개 페이지 접근 가능
로그인 시 전체 페이지 접근 가능 (일부 기능은 권한 필요)
소개 페이지 (동아리 소개, 읽은 책 리스트, 표지, 사진)
게시판 기능
자료실 기능
사진 갤러리 기능
공지 기능
캘린더 표시 기능
캘린더 모임 일정 표시 기능
모임 일정 등록/수정 기능 (관리자, 도서관 사서만 가능)
회원 연락처 정보 조회 기능 (로그인 후 가능)

회원 삭제 기능(도서관 사서만 가능)

회원 메일 주소 클릭 시 메일 보내기 기능
한 줄 수다방 기능 (글 등록, 댓글 등록 - 3줄 이내)
신규 회원 가입 시 관리자 이메일([00@gmail.com]) 알림 기능
회원 정보 표시 형식: '실명 (아이디)'

4.2. 제외 범위 (Out-of-Scope)

비밀번호 찾기/변경 기능 (모든 비밀번호가 고정)
회원 탈퇴 기능
결제 시스템 연동
실시간 채팅 기능 (수다방 제외)
푸시 알림 기능 (이메일 알림 외)
파일 업로드/다운로드 용량 제한 설정 (초기 버전)

5. 기능 요구사항 (Functional Requirements)

5.1. 회원 관리 (User Management)

FR-UM-001: 회원 가입
사용자는 이름, 아이디, 휴대폰 번호, 메일 주소를 필수로 입력하여 가입 신청할 수 있다.
가입 신청 시, 비밀번호 입력란에 'test123'을 입력해야만 다음 단계로 진행된다. (실제 비밀번호 저장 기능은 없으나, 가입 절차상의 필수 입력 값으로 사용)
가입 완료 시, 해당 사용자는 '승인 대기' 상태가 된다.

FR-UM-002: 관리자 승인 대기 알림
새로운 회원이 가입 신청을 완료하면, 관리자 이메일([00@gmail.com])로 신규 가입자 발생 알림 메일이 전송된다.

FR-UM-003: 관리자 회원 승인/반려
관리자는 관리자 페이지에서 '승인 대기' 상태의 회원 목록을 확인할 수 있다.
관리자는 각 대기 중인 회원을 '승인'하거나 '반려'할 수 있다.
승인된 회원은 정상적으로 로그인할 수 있게 된다.
반려된 회원은 로그인할 수 없으며, 가입 상태는 '반려' 또는 '삭제' 상태로 변경된다.

FR-UM-004: 로그인
승인된 일반 회원 및 도서관 사서는 아이디와 비밀번호 'test123'으로 로그인할 수 있다.
관리자는 아이디 'admin'과 비밀번호 'admin'으로 로그인할 수 있다.
로그인 성공 시 사용자의 권한(일반 회원, 도서관 사서, 관리자)에 따라 페이지 접근 권한이 부여된다.
로그인 실패 시 적절한 오류 메시지를 표시한다.

FR-UM-005: 회원 정보 수정 (관리자와 회원 본인)
관리자는 관리자 페이지에서 가입된 모든 회원의 개인 정보(이름, 아이디, 휴대폰 번호, 메일 주소)를 수정할 수 있다.
관리자는 회원의 구분(동아리 멤버, 도서관 사서)을 수정할 수 있다.

FR-UM-006: 회원 정보 삭제 (도서관 사서)
도서관 사서로 구분된 사용자는 회원 정보를 삭제할 수 있다.
회원 정보 삭제 시, 한 번 더 비밀번호(test123)를 확인한다.

FR-UM-007: 회원 정보 표시
홈페이지 내에서 사용자의 이름은 '실명 (아이디)' 형식으로 표시된다. (예: 홍길동 (hong.g))

5.2. 콘텐츠 관리 (Content Management)

FR-CM-001: 소개 페이지
로그인하지 않은 사용자도 접근할 수 있다.
동아리 소개 글, 지금까지 읽은 책의 리스트, 읽은 책의 표지 이미지, 동아리 활동 사진 몇 장을 포함한다.

FR-CM-002: 게시판
로그인한 사용자만 접근할 수 있다.
글 목록 조회, 글 상세 조회, 글 작성, 글 수정, 글 삭제 기능을 제공한다.

FR-CM-003: 자료실
로그인한 사용자만 접근할 수 있다.
자료 목록 조회, 자료 상세 조회, 자료 업로드, 자료 수정, 자료 삭제, 자료 검색 기능을 제공한다. (첨부 파일 기능 포함)

FR-CM-004: 사진 갤러리
로그인한 사용자만 접근할 수 있다.
사진 목록 조회, 사진 상세 조회, 사진 업로드, 사진 수정, 사진 삭제 기능을 제공한다.

FR-CM-005: 공지
로그인한 사용자만 접근할 수 있다. (관리자와 도서관 사서는 작성/수정/삭제 가능)
공지 목록 조회, 공지 상세 조회, 공지 검색 기능을 제공한다.
관리자와 도서관 사서는 공지를 작성, 수정, 삭제할 수 있다.

5.3. 캘린더 및 일정 관리 (Calendar & Schedule Management)

FR-CS-001: 캘린더 표시
홈페이지 메인에 캘린더 위젯이 표시된다.
캘린더에는 등록된 모임 일정이 특정 방식으로 표시된다.

FR-CS-002: 모임 일정 표시
캘린더에 등록된 모임 일정이 표시된다.
일정 클릭 시 상세 정보(제목, 날짜, 시간, 장소 등)를 확인할 수 있다.

FR-CS-003: 모임 일정 등록/수정 (관리자, 사서)
관리자 또는 도서관 사서 권한을 가진 사용자만 모임 일정을 등록, 수정, 삭제할 수 있다.
일정 등록 시 제목, 날짜, 시간, 장소, 내용 등을 입력할 수 있다.

5.4. 소통 기능 (Communication Features)

FR-SC-001: 회원 연락처 정보 조회
로그인한 사용자만 접근할 수 있다.
가입된 회원의 이름(아이디), 휴대폰 번호, 메일 주소 목록을 조회할 수 있다.

FR-SC-002: 메일 보내기 기능
회원 연락처 목록 또는 상세 정보에서 회원의 메일 주소를 클릭하면 기본 메일 클라이언트가 실행되어 해당 주소로 메일을 보낼 수 있도록 연결된다. (mailto: 링크)

FR-SC-003: 한 줄 수다방
로그인한 사용자만 접근할 수 있다.
3줄 내외의 짧은 텍스트 메시지를 작성하여 등록할 수 있다.
다른 사용자가 등록한 수다 글에 댓글을 달 수 있다.
수다방 목록에는 수다 글과 해당 글에 달린 댓글이 함께 표시된다.

5.5. 관리자 기능 (Administrator Features)

FR-AD-001: 관리자 페이지
관리자 권한으로 로그인한 사용자만 접근할 수 있다.
회원 가입 승인 목록 및 기능 제공 (FR-UM-003 상세)
회원 정보 조회 및 수정 기능 제공 (FR-UM-005 상세)
모임 일정 관리 기능 접근 (FR-CS-003 상세)
회원 정보 삭제 기능 (FR-UM-006 상세)

6. 비기능 요구사항 (Non-Functional Requirements)

NFR-SE-001: 보안 (Security)
로그인되지 않은 사용자의 특정 페이지 접근을 통제해야 한다.
관리자 페이지는 관리자 권한 사용자만 접근할 수 있어야 한다.
고정된 비밀번호 사용으로 인해 보안 취약점이 존재하므로, 해당 정책은 향후 개선될 수 있다. (현재 요구사항 준수)

NFR-US-001: 사용성 (Usability)
직관적인 네비게이션 및 레이아웃을 제공하여 사용자가 쉽게 원하는 정보에 접근할 수 있도록 한다.
모든 기능은 웹 표준을 준수하여 일반적인 웹 브라우저에서 정상적으로 작동해야 한다.
반응형 웹으로 구현되어 모바일 페이지에서도 잘 표시되고 정상적으로 작동해야 한다.

NFR-PE-001: 성능 (Performance)
페이지 로딩 시간은 일반적인 웹 환경에서 허용 가능한 수준이어야 한다.
게시물, 자료, 사진 목록 조회 시 과도한 지연이 발생하지 않아야 한다.

NFR-CO-001: 호환성 (Compatibility)
주요 데스크톱 및 모바일 웹 브라우저에서 정상적으로 작동해야 한다.

7. 제약사항 (Constraints)

일반 회원 및 사서의 로그인/가입 비밀번호는 'test123'으로 고정된다.
관리자 아이디는 'admin', 비밀번호는 'admin'으로 고정된다.
신규 가입 알림은 '[00@gmail.com]'으로만 발송된다.
홈페이지 내 사용자 이름 표시는 '실명 (아이디)' 형식으로 강제된다.
한 줄 수다방 게시글은 3줄 이내로 제한된다.

8. 가정 (Assumptions)

사용자는 기본적인 웹사이트 사용 방법을 알고 있다.
관리자는 제공된 관리자 계정으로 시스템 관리를 수행한다.
이메일 발송 기능은 시스템 환경에 의해 지원된다.

9. 향후 고려사항 (Future Considerations)

비밀번호 변경/찾기 기능 추가 (보안 강화)
모임 일정 알림 기능 (이메일, SMS 등)
게시판 댓글 기능 강화 (대댓글 등)
본 PRD는 제공된 요구사항을 기반으로 작성되었으며, 프로젝트 진행 중 추가 논의 및 협의를 통해 상세화될 수 있습니다.
